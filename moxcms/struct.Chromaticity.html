pub const fn to_scaled_xyzd(&self) -> Xyzd

Get the color representation with component sum 1.

In contrast to the XYZ representation defined through setting Y to a known value (such as 1 in Self::to_xyz) this representation can be uniquely derived from the xy coordinates with no ambiguities. It is scaled from the original XYZ color by diving by X + Y + Z. Note that, in particular, this method is well-defined even if the original color had pure chromamatic information with no luminance (Y = 0) and will preserve that information, whereas Self::to_xyz is ill-defined and returns an incorrect value.
