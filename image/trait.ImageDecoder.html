fn read_image(self, buf: &mut [u8]) -> ImageResult<()>
where Self: Sized,

Returns all the bytes in the image.

This function takes a slice of bytes and writes the pixel data of the image into it. buf does not need to be aligned to any byte boundaries. However, alignment to 2 or 4 byte boundaries may result in small performance improvements for certain decoder implementations.

The returned pixel data will always be in native endian. This allows [u16] and [f32] slices to be cast to [u8] and used for this method.

ยงPanics

This function panics if buf.len() != self.total_bytes().

ยงExamples
fn read_16bit_image(decoder: impl ImageDecoder) -> Vec<u16> {
    let mut buf: Vec<u16> = vec![0; (decoder.total_bytes() / 2) as usize];
    decoder.read_image(bytemuck::cast_slice_mut(&mut buf));
    buf
}

fn total_bytes(&self) -> u64

Returns the total number of bytes in the decoded image.

This is the size of the buffer that must be passed to read_image or read_image_with_progress. The returned value may exceed usize::MAX, in which case it isnโt actually possible to construct a buffer to decode all the image data into. If, however, the size does not fit in a u64 then u64::MAX is returned.

fn set_limits(&mut self, limits: Limits) -> ImageResult<()>

Set the decoder to have the specified limits. See Limits for the different kinds of limits that is possible to set.

Note to implementors: make sure you call Limits::check_support so that decoding fails if any unsupported strict limits are set. Also make sure you call Limits::check_dimensions to check the max_image_width and max_image_height limits.

Note: By default, no limits are defined. This may be changed in future major version increases.
